
# LangGraph 활용 아이디어 (Future Works)

이 문서는 프로젝트의 향후 발전 방향에 대한 아이디어를 기록합니다.

### 1. 개별 맞춤형 피드백 분기 (Branching)
사용자의 코드 상태에 따라 서로 다른 분석 노드로 이동하여 맞춤형 처방을 제공합니다.

- **[분기 1] 틀린 코드 (Logic Error)**
  - → **`Refactor` 생략**
  - → **`Counter-Example Node` (반례 생성)**: "이 입력값을 넣으면 오답이 나옵니다"라고 구체적으로 알려줌.
- **[분기 2] 비효율적 코드 (Inefficient)**
  - → **`Hint Node` (힌트 제공)**: 정답 코드를 바로 주지 않고, "이중 루프 대신 해시맵을 써보세요" 처럼 방향만 제시.
- **[분기 3] 완벽한 코드 (Perfect)**
  - → **`Advanced Node` (심화 학습)**: "이 문제를 다른 언어로 푼다면?", "더 어려운 유사 문제 추천".

### 2. 소크라테스식 튜터링 (Human-in-the-loop)
단방향 분석이 아니라, 대화를 통해 스스로 답을 찾도록 유도합니다.

- **시나리오**:
  1. 사용자 코드 제출.
  2. AI: "논리는 맞는데 시간 복잡도가 O(N^2)네요. 줄일 방법이 없을까요?" (질문 후 대기 - `interrupt`)
  3. 사용자: 코드 수정 후 재제출.
  4. AI: (이전 State와 비교) "네, 정렬을 먼저 하니 훨씬 빨라졌네요! 잘하셨습니다."
